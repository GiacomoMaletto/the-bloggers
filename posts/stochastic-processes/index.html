What's a stochastic process?

We want to define a stochastic (or random) process. We can consider the classical example in probability theory: tossing a coin. Let’s suppose that the probability of obtaining head is $p$, whereas the once of obtaining tail $ 1 - p$, as a consequence. Well, let’s think of tossing the coin more and more times in a succession of independent trials. We can model each trial as a Bernoulli random variable. But if we want to model the all succession, we need a new mathematical object: a random process. So we have just examined the first example of discrete-time random process. It looks like a succession of independent and identically distributed random variables, but we mustn’t fall into the following trap: a stochastic process is much more general than thus, in fact the time is not necessarily discrete, and the variabiles don’t have to be independent.

Before giving the definition let’s do another example, in which the time is not discrete. Let’s suppose to count the number of customers who are queuing in front of a server at the post office. So the number of customers at the istant $t$ is random, and we call it $X(t)$. Then if we consider a bounded interval of time $[a,b]$ then we $X(t)$ is a random variable for all $x \in [a,b]$. Therefore the collection of random variables $\{X(t), t \in [a,b]\}$ is a continous-time random process.

Now, we want to give a general definitios. It’s clear that we are talking about random variables, so we need a probability space $(\Omega, \Im, \mathbb{P})$, where $\Im$ is a $\sigma$-algebra on $\Omega$ and $\mathbb{P}$ is a probability measure. In the first example we can see that we have a random variable for each $n \in \mathbb{N}$. So we could say that a discrete random variable is a function $X: \Omega \times \mathbb{N} \to K $ so that $X(\cdot, n) \to K $ is measurable function for each $n$, where $\Omega$ is probability space and $K$ is the codomain of each random variable. But this defintion doesn’t include the second example.

The following is one of the best definitions of random process, according to me. But I point out that it’s not tha most general as possible, because it doesn’t contemplate the circumstance that the random variables are not defined on the same probability space. But it’s also important to underline that this eventuality is not so important for applications.

Definition (Stochastic process):

Let $(\Omega, \Im, \mathbb{P})$ be a probability space. Let $K$ be a set, called index set, and $S$ a measurable space, called state space. Then the function $X: \Omega \times K \to S $ is called $S$-valued random process if and only if $X(\cdot, k)$ is measurable for each $k \in K$.

We could say that a $S$-valued stochastic process is a collection of $S$-valued random variables. So it’s a collection $\{X(t), t \in K\}$ indexed by $K$. Now, if $K$ is at the most countable, I have a dicrete-time process, otherwise a continous-time one. Then, if $S$ is at the most countable, I have a discrete space process, otherwise a continous-space one.

Now, we have just given the definition of stochastic process. Then they can have particular properties: being markovian or semi-markovian, for example, or being couting processes. But how can we take advantage of them? Fisrt of all, the most of the random experiences are not isolated events, but collections of them (let’s think of a particle moving in a fluid, or of the price of a stock’s share); so a random process is the best model for some real situations. Then, thanks to limit theorems, stochastics processes usually have limit behaviours, which can be interpreted as the expected outcome of a random phenomenon.
